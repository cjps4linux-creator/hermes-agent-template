### What you get

- **Agent Contract Template** (`agent/agent.spec.md`) — mandatory schema for identity, mission, scope, inputs, outputs, constraints, dependencies, tools, memory, failure taxonomy, observability, and evaluation.
- **Runnable Agent Profile** (`agent/AGENT.md`) — drop-in execution profile your runtime can load directly.
- **Contract Linter** (`scripts/validate_agent_contract.py`) — Python 3 validator that enforces the schema before your agent ships.
- **New-Agent Scaffold** (`scripts/skeleton.sh`) — generate a new agent folder from the template in under 30 seconds.
- **Pre-Publish Checklist** (`docs/checklist.md`) — review input validation, tool permissions, memory retention, observability, and versioning before release.
- **Migration Notes** (`docs/migration.md`) — adapt the template to your existing LLM backend or runtime.
- **Example Agent** (`example-agent/`) — a complete Contract Review Assistant with `SOUL.md`, `contract.md`, and a README-ready workflow.
- **Commercial License** — MIT, commercial use allowed.

### Who this is for

- Teams shipping multi-agent systems who need stable contracts.
- Engineers migrating prompt wrappers into production-grade agents.
- Founders who need repeatable agent design, not prompt roulette.
- Operators who want agents that fail predictably and are safe to operate.

---

### Why this template

Most AI agents shipped today are prompt wrappers with no contract, no failure taxonomy, and no observable health checks. This template turns scattered prompts into auditable, testable, evaluatable systems.
